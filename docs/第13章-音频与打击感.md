## 13.1 音乐

### 13.1.1 风格

走**国风器乐**,以**古琴**为主心骨——古琴是文人的乐器,稀疏、内省,与"书卷气"严丝合缝;再配古筝、笛箫、琵琶,战斗段落加战鼓。

### 13.1.2 贴主题的动机:褪色 → 还原

让音乐与"文字崩坏"主题同构:崩坏的、早期的区域,配乐稀疏、微微失谐、空洞;随着玩家正字、深入,音乐"补全"回来,变得完整温润。**配乐本身也在被玩家修复。**

### 13.1.3 现实的曲目规模

个人开发不可能做二十首。务实的清单:

- 主菜单 1 首
- 战斗 3~4 首(低/中/高强度,跨章节复用)
- Boss 1 首、终 Boss 1 首
- 地图氛围 1 首
- 胜负短曲若干

合计约 **8~12 首**。

一个省料技巧:做**分层自适应音乐**——一首基底,战斗升温时叠入鼓与琵琶声部,用更少的素材撑出更多变化。

### 13.1.4 来源

- AI 音乐工具(可出古风器乐,个人可用,但**商用授权务必逐一核实**)。
- 免费国风曲库。
- 预算允许就找自由音乐人小批量定制(质量最高)。

## 13.2 音效

核心洞察:**整个音效世界,就是一间书房的声音**——墨、笔、纸、印。没有一个泛用的"奇幻游戏音效"。

| 元素 | 音效 |
|---|---|
| 拆 | 研墨声 / 纸面干裂的脆响——字被"研开" |
| 合 | 笔锋走纸的"唰"声,收尾一个利落的顿笔——字被"写出来" |
| 出字 · 火 | 火苗舔纸的噼啪 |
| 出字 · 水 | 墨滴入水的流动 |
| 出字 · 金 | 刀锋出鞘的金属嗡鸣 |
| 出字 · 木 | 木裂叶响 |
| 出字 · 心 | 一记低沉发寒的钟鸣 |
| 印章 | 印章"咚"地盖在纸上——用于 UI 确认、领取奖励 |
| 字怪 | 墨变成的怪物——湿黏、咕哝、扭曲的"乱语" |

**两个关键点**:

- **逐笔触声**:配合逐笔书写的程序动画,每一笔都对一个笔触声——这个同步是巨大的体验加分。
- **崩坏失真动机**:被腐化的东西(字怪、讹)声音要微微"不对劲"——失谐、轻微故障感、倒放;正字之后声音"解析"回干净的清音。**音效也在镜像核心主题。**

音效对个人开发很友好:笔、墨、纸、印章都是能自己用手机麦克风录的实物拟音(foley),再补免费音效库与 AI 音效工具即可。

## 13.3 打击感

打击感是动作"砸实"的爽感。回合制卡牌虽不像动作游戏,但原理一样——它是一个**在动作命中瞬间同时炸开的多感官捆绑包**。

### 13.3.1 多感官捆绑包

| 层 | 内容 |
|---|---|
| 视觉冲击 | 命中时墨迹炸开;敌人受击抖动、缩放回弹、瞬白闪;震屏幅度随伤害缩放;伤害数字"弹"出来(先放大再回落,按属性配色) |
| 音效冲击 | 音效精准落在命中那一帧——分层:尖锐瞬态(命中点)+ 属性质感 body + 尾音 |
| 触觉 | 手机震动。小击轻震、重击重震。这是移动端极廉价又极有效的一层手感 |
| 微停顿(顿帧) | 重击瞬间让一切冻结 80~120 毫秒——脆爽手感的真正秘密,实现成本几乎为零 |

击杀或暴击可再加一点慢动作。

### 13.3.2 签名时刻

几个高光时刻要专门做反馈:

- **合出一个字**:逐笔书写 + 笔声 + 收尾印章"咚"——因为时刻都在合字,它必须每次都爽。
- **升阶**(火→炎→焱→燚):逐级更响更亮。
- **击杀字怪**:字怪"崩解"回散落的墨笔画,一个"正确"的字一闪而过(你把它"正"回来了),既爽又扣题。
- **斩杀**(锋锐流处决):一记决断的刀光 + 金属脆响 + 短暂冻结。

### 13.3.3 关键的好消息

上面整个打击感清单——震屏、顿帧、粒子、震动、数字弹跳、时序——**几乎全是代码,Claude Code 全能写,不花一分美术钱。**

打击感是个人开发者**最可控**的一块,值得反复打磨。

## 13.4 跨端特性(v0.3)

- **音频可选化设计**:Steam 玩家戴耳机为主(音效可以做厚做满),但移动端玩家常静音(地铁、办公室)。所以**视觉与触觉反馈必须能独立扛起打击感**,任何关键信息绝不能只靠声音传达。
- **包体**:音频是体积大户——移动端要狠压缩、音乐流式 / 懒加载、音效短促;Steam 端可以追加高保真版本作为额外资源包(可选下载)。
- **触觉反馈**:把振动 / 触觉调用封进平台适配层(见第 16 章 v0.3)。Steam 走 Steam Input(手柄振动);iOS 走 Haptic Engine;Android 走系统振动 API。统一一个接口。

## 13.5 与其他系统的关系

- **美术与 UI(第 12 章)**:程序动效与音效、震屏共同构成打击感。
- **核心玩法(第 3 章)**:合字、出字、升阶的反馈节点对应三动词。
- **技术架构(第 16 章)**:振动 API 由平台适配层抽象。
- **变现(第 14 章)**:音频可选化设计同时服务静音玩家体验。
