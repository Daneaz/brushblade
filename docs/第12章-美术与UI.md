## 12.1 美术风格

### 12.1.1 核心理念:字即画

最重要的一个判断:**这个游戏的内容本身就是文字。**

字卡的"画",很大程度上**就可以是那个汉字本身的精美书法**——这不是插画问题,是排版与渲染问题。

> 五百个字,你不需要画五百张插画,你需要把五百个字用漂亮的书法呈现出来。

它把美术工作量从"插画工程"砍成"字体 + 少量配图",并直接复用书法皮肤那条变现线(见第 14 章)。**这是整个美术方向的地基。**

### 12.1.2 风格定调:褪色的世界,正字还原色彩

不做泛泛的"水墨风",给它一个具体的钩子:

> 因为文字在崩坏,整个字界是墨色褪去、灰败失色的;玩家每"正"一个字,色彩就回来一分。

颜色 = 力量 = 修复。世界是水墨,而魔法是色。这条设定让美术与剧情严丝合缝。

### 12.1.3 配色

| 用途 | 颜色 |
|---|---|
| 底色 | 宣纸米 + 浓淡墨 |
| 唯一点缀色 | 朱砂红(印章的颜色) |
| 五行五色 | 火朱、水青、木绿、金赭、心黛紫——**只出现在字本身与技能特效上** |

世界保持水墨;字一发光就是战斗的"读盘"线索。

### 12.1.4 视觉母题

三个反复出现的母题:**印章**(按钮、确认、法宝边框)、**竹简**(字库陈列)、**卷轴**(菜单与地图展开)。墨迹的"飞白"作为特效肌理。

### 12.1.5 字怪

字怪 = 妖魔化的汉字——墨色凝结成的"字兽",半是字形半是凶相。

这是一个**统一的 prompt 家族**(都是"墨变成的字怪"),AI 批量出图时风格极易保持一致,远比画一百个互不相干的怪物省力。

### 12.1.6 动效:靠程序,不靠帧动画

给个人开发的关键忠告——**不做骨骼 / 帧动画。**

"打击感"全部交给:墨迹飞溅、墨色晕染、字形辉光、粒子飞白、震屏,以及一个特别契合的效果——**字"一笔一划写出来"的笔画动画**(用 SVG 路径或 shader 程序实现,不是手画帧)。

字怪用静态图 + 简单 tween(轻微浮动、受击抖动、缩放)即可。这些 Claude Code 全能写,无需任何动画师。

## 12.2 UI / 界面

### 12.2.1 屏幕方向(v0.3 双端方案)

**PC 端:横屏**(16:9 / 16:10,Steam Deck 16:10);**移动端:竖屏**,纯触控。

布局思路:**核心交互区(砚台)在屏幕底部,敌人区在屏幕顶部,中部留给战斗演出**——这个垂直分层在横屏和竖屏下都成立,差别只是:

- **横屏**(PC):敌人横排展开,字库/部件分两行排在底部,中间演出区更宽阔。地图、字典等次级界面可用左右分栏或全屏弹层。
- **竖屏**(移动):敌人竖叠,字库/部件压紧底部三分之一,演出区相对窄但更聚焦。

> "卷轴、竹简、印章"这套母题在横竖屏下天然伸缩——这是选这套视觉语言的隐藏好处。一套核心 UI 组件 + 横竖屏布局变体即可双端发布。

### 12.2.2 战斗界面(核心)

这是最难的一屏,因为拆/合/出字是新交互。给它一个直觉隐喻——**砚台**。

布局(自上而下):

- **顶部**:1~3 只字怪,各显示生命、护甲、心魔/灼烧等状态图标,以及一个弱点部首标识。
- **中部**:战斗演出区(墨迹特效在此炸开)。
- **玩家状态条**:生命,以及本回合 3 点行动点(画成 3 盏灯 / 3 点墨)。
- **底部 · 砚台工作台**:你的字(竹简卡)在一排,散件部件在另一排。

砚台隐喻下的交互:

| 动作 | 操作 |
|---|---|
| 出字 | 点字 → 向敌人施放 |
| 拆 | 把字拖进砚台 → 字被"研开"化为部件飞入池中 |
| 合 | 点选池中部件 → 砚台即时弹出**合字提示**("可合成:焚、林……")→ 点确认锻造 |

砚台这个隐喻很妙:拆 = 把字研磨回墨,合 = 用墨写出新字。新手一看就懂。**合字提示面板必须显眼**——它是防卡手的命门。

### 12.2.3 地图界面

5 个章节做成一幅**横向长卷**,随进度徐徐展开,每章是画卷的一段。节点是卷上的墨点与印章。这是最美也最贴主题的一屏。

### 12.2.4 字典图鉴

meta 收集界面做成一本真正的**字帖**——你正过的每个字、收服的每只字怪,逐页填进去。收集驱动 + 文化质感一举两得。

### 12.2.5 其他界面

主界面(卷轴展开式)、奖励三选一、字库/部件管理、结算界面。

### 12.2.6 主菜单与系统界面(v0.3 极简版)

Premium 买断模式下,养成 / 抽卡 / 月卡 / 商店 / 任务 / 实名 全部不存在。主菜单与系统层只剩:

- **主菜单**:卷轴展开式,选项极少——开始新局 / 继续 / 字典图鉴 / 难度 / 设置 / 退出。
- **设置**:音量、画面、语言(预留)、键鼠映射(PC)、触控灵敏度(移动)、退出登录(若有 Steam 账户绑定)。
- **难度选择**:首通后解锁。书法段位列表 + 当前段位累计变异规则展示。
- **存档管理**:Steam Cloud 自动同步;移动端本地存档 +(可选)Apple/Google 账号云同步。
- **字典图鉴**:已正过的字、已收服的字怪、已收集的法宝,逐页填进字帖。
- **结算界面**:本局战绩(用字、流派、关卡、死因)+ 鼓励再来一局。

> **关键视觉戒律**:整体水墨基调贯穿到底,**任何鲜艳的霓虹/赛博/氪金感视觉一律禁止**——这是 Premium 独立游戏的气质底线,玩家会因为这点视觉一致性而尊重你。

### 12.2.7 可读性原则(双端)

- 当前出战的字形要够大(它就是玩法本体)。
- 敌人状态全部用清晰图标(灼烧 = 火苗、心魔 = 墨点、护甲 = 盾、冻结 = 冰)。
- 颜色克制使用,靠属性辉光来"读"战局。
- 任何字点一下显示拼音与释义。

### 12.2.8 双端适配(v0.3)

- **Steam(PC/Mac)**:横屏 16:9 / 16:10 为主,**完整支持鼠标 + 键盘热键**(数字键打字、Tab 切目标、Esc 暂停)。Steam Deck 兼容性测试为必做项(见 LAUNCH_CHECKLIST 阶段 5)。
- **iOS / Android**:竖屏触控,适配刘海与异形屏的安全区。一手拇指操作要全程成立。
- **共享 UI 组件**:卷轴/竹简/印章这套母题在横竖屏下天然伸缩,横屏拉宽,竖屏拉高,组件不变。

> 不做横竖屏切换(PC 不需要竖屏,移动竖屏锁定)——这能省掉一整套自适应逻辑,把工作量留给真正能感知到差别的内容上。

## 12.3 与其他系统的关系

- **核心玩法(第 3 章)**:战斗界面的砚台隐喻把三动词交互可视化。
- **拆合引擎(第 4 章)**:合字提示面板是引擎提示功能的 UI 呈现。
- **变现(第 14 章 v0.3)**:Premium 模式下不再有皮肤变现一线;但「字即画」仍是美术减负的关键。
- **音频与打击感(第 13 章)**:程序动效与音效、震屏共同构成打击感。
- **技术架构(第 16 章 v0.3)**:无帧动画的选择降低了包体与性能压力,Steam Deck 兼容性也容易过。
