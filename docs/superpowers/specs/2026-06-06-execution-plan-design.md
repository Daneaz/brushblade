# 《字·斗》执行计划设计文档

> 创建日期：2026-06-06
> 状态：设计阶段（待实现计划）
> 引擎：Cocos Creator 3.x（TypeScript）
> 首发范围：3 流派（燎原/江海/锋锐）+ 3 章 + 5 难度阶梯，Steam 优先

---

## 核心原则

- **双轨并行**：用户任务（账号、美术、社区）与 Claude Code 任务（数据管线、编码）同时推进，互不阻塞。
- **分阶段 go/no-go 门**：每阶段结束前有明确验收标准，不通过不进入下一阶段。
- **浏览器 demo 替代纸面原型**：Phase 1 由 CC 构建可在浏览器运行的战斗 demo，用户在 demo 中验证核心手感，替代纸片原型。

---

## 整体时间线

```
Phase 0 (2~3周)
  [用户] 注册账号 / 选字体 / 确认定价
  [CC]   数据管线脚本 / 项目骨架 / CLAUDE.md / 配方校验工具

Phase 1 (1~2周)
  [CC]   浏览器战斗 demo
  [用户] 玩 demo → 拍板配方深度(Q1) → go/no-go

Phase 2 (2~3月)
  [CC主] 拆合引擎 → 战斗系统 → 燎原流 → UI → 打击感 → 双端真机
  [用户] 精筛字表 / 逐字设计战斗效果数值 / 决定讹字机制(Q2)

Phase 3 ∥ Phase 4 (各2~3月，并行)
  [CC]   3章内容 / 3流派 / 法宝 / 引导 / 难度阶梯
  [用户] 商店页 / 宣传片 / 封测 / 社区运营

Phase 5 (1月)
  [用户+CC] 正式上架：Steam Deck验证 / 审核提交 / 发布公告
```

## Phase 1：浏览器战斗 Demo（1~2 周）

### CC 构建内容

纯 HTML/CSS/JavaScript，无需安装，本地浏览器打开即可玩。

| 模块 | 内容 |
|---|---|
| 字集 | ~30 个字：火系 8 个（火炎焱焚燃烧灯灰）/ 水系 6 个 / 木系 6 个 / 金系 6 个 + 关键部件 |
| 战斗框架 | 玩家 HP 50、AP 3/回合、部件池（12 槽）、字库（8 槽） |
| 三动词 | 拆 / 合 / 出字，全部可交互操作 |
| 提示引擎 | 实时显示"当前部件池能合成什么" + "差一步提示" |
| 敌人 | 1 只木属性小怪 + 1 只小 Boss（测试连招节奏） |
| 配方深度 AB | 可切换"一步合成"vs"两步合成"（用于 Q1 对比） |
| 胜败条件 | 玩家 HP 归零 = 失败；Boss 倒下 = 胜利 |

### 用户验证

玩 5~10 局后回答：
1. 拆/合/出字的决策有趣吗？
2. 一步合成 vs 两步合成，哪个更爽？
3. 什么时候最无聊/卡手？

### Phase 1 go/no-go 门槛

- ✅ 通过：核心手感成立，Q1（配方深度）拍板 → 进入 Phase 2
- ❌ 不通过：调整机制后重测，或停止项目

## Phase 2：垂直切片（2~3 月）

### CC 执行顺序（串行，有依赖）

```
1. 拆合规则引擎（TypeScript 纯逻辑，基于配方 DAG）
2. 战斗状态机 + 回合循环
3. 提示引擎（配方 DAG 实时查询，增量重算）
4. 燎原流完整字表 + 3~5 件法宝
5. 1 只字怪 + 1 只小 Boss（AI 行为）
6. 基础 UI（字库 / 部件池 / AP / HP / 敌人面板）
7. 打击感最小集（震屏 + 命中停顿 + 1 个核心音效）
8. Cocos 导出 Windows/Mac 可执行文件，本地自测通过
9. iOS/Android 真机自测一关跑通
```

### 用户同期任务

| 任务 | 说明 |
|---|---|
| 精筛候选字表 | 从脚本输出的 ~500 字中圈定最终 MVP 字表（~200 字） |
| 逐字设计战斗效果与数值 | "焚打 18 点 AOE"等效果只能人工决策 |
| 决定 Q2 | 讹字机制是否纳入首发（否则留作 DLC 1） |

### Phase 2 验收标准

- 燎原流跑通 1 关（5 场战斗 + 1 小 Boss）
- Windows/Mac 可执行文件自测通过
- iOS/Android 真机可运行且帧率流畅

## Phase 5：正式上架（约 1 月）

### 用户任务

**Steam：**
- 发布日期确定 + Coming Soon 倒计时配置
- 首发折扣设定（建议 10%）
- Steam Deck 兼容性验证（Proton 测试）
- 国区定价确认（约美区 50%，即 ¥45）
- Steam 成就/收藏品/集换式卡牌（可选）

**iOS/Android：**
- App Store 应用页（中英文截图 + 描述 + 隐私问卷）
- Google Play 应用页（同上）
- 提交审核（预留 1~2 周缓冲）

**通用：**
- 隐私政策 + 用户协议英文终稿
- 客服/反馈邮箱配置
- 发布前一周公告 → 各社区同步

### CC 任务

- Steam Deck 输入适配（键鼠/手柄/Steam Input 适配层）
- i18n 框架预留（文本与数据分离，便于后续补英文/日文）
- 上架前 Bug 修复 + 性能优化

### Phase 5 验收标准

- Steam 页面审核通过，iOS/Android 审核提交
- 游戏在 Steam Deck 上标记为兼容
- 隐私政策 + 用户协议英文版在位
