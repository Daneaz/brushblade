## 3.1 战斗概述

《字·斗》的战斗是**回合制单人战斗**。玩家(正字者)在每一关面对 1~3 只「字怪」,双方轮流行动,直到一方倒下。

战斗不是动作博弈,而是**资源管理 + 组合构筑**:玩家在有限的行动点内,通过"拆 / 合 / 出字"三个动词运转自己的字库,打出由汉字字义决定的战斗效果。

战斗的乐趣来源:
1. **构筑的爽感**——把零散的部件重组成强力的字。
2. **取舍的紧张**——拆还是留、合还是出,每个决策都有机会成本。
3. **滚雪球的高潮**——后期 build 成型时一套连招清场。

---

## 3.2 三个核心动词

战斗的全部操作收敛为三个动词。每回合玩家用行动点(AP)自由组合它们。

### 3.2.1 出字(Cast)

打出字库中的一个汉字,触发由其字义 / 部首决定的战斗效果。

- 效果类型:单体伤害、全体伤害(AOE)、治疗、护盾、召唤、施加 debuff、施加增益等。
- 目标:伤害类需指定敌人(或自动索敌),AOE / 治疗 / 自身增益无需选目标。
- 出字后,该字进入"已使用"状态,**本场战斗内不再可用**(除非有法宝/效果让其回到字库)。
- AP 消耗:基础字 1 AP,高阶字(焱、燚、剑、海、念等)2 AP。

### 3.2.2 拆(Dismantle)

将字库中的一个汉字拆成它配方中的原料部件,丢进**部件池**。

- 拆解依据该字的"游戏配方"(见第 4 章)。例:拆「焚」→ 得「林」+「火」。
- 基础规则:**无损返还**——配方里的全部原料都进部件池。损耗或加成由法宝调整。
- 独体字(火、水、木、日、月等)本身即部件,**不可再拆**。
- AP 消耗:1 AP。

### 3.2.3 合(Compose)

消耗部件池中的散件,锻造出一个新汉字,放入字库。

- 当部件池(+ 字库中的低阶字,原料优先取池)包含某汉字配方的**全部原料**时,即可锻造该字。
- 锻造出的字进入字库,当回合即可出字(若 AP 足够)。
- 只有"正字"(真实的、被设计过的汉字)可被锻造满威力;乱凑的部件最多合出虚弱的"讹字"(见第 4 章三层系统)。
- AP 消耗:1 AP。

## 3.3 战斗资源:行动点(AP)

- 每回合开始,玩家获得 **3 点行动点**。
- 行动点不可跨回合保留(除非法宝改变)。
- 典型的一个回合 = "拆一个 + 合一个 + 出一个" = 3 个 1-AP 动作。
- 高阶字出字消耗 2 AP,因此打出一个大招的回合,只能再做一个其他动作。
- 法宝「兵贵神速」可将每回合 AP 提升到 4。

行动点是战斗的核心节奏阀门:它逼迫玩家在"这回合是攒资源(拆/合)还是输出(出字)"之间持续做决策。

---

## 3.4 部首即属性

战斗属性不另行设计,直接借用汉字部首天然编码的语义。玩家对"这个字什么意思"的直觉,就是它的战斗效果。

| 部首 | 属性 | 战斗倾向 | 典型字 |
|---|---|---|---|
| 火 / 灬 | 火 | 高爆发、持续灼烧 | 炎、焰、烈、焚 |
| 氵 / 水 / 冫 | 水 | 治疗、续航、冰冻 | 治、润、海、冰 |
| 木 / 艹 / 竹 | 木 | 召唤、成长 | 林、森、苗、树 |
| 钅 / 刂 / 刀 / 戈 | 金 | 穿透、破甲 | 刃、剑、锋、斩 |
| 土 / 山 / 石 | 土 | 护盾、防御(跨流派工具字家族,不成流派) | 圭、埋、壁、堡 |
| 心 / 忄 | 心 | 控制、debuff | 惧、惑、慑、念 |

六属性 = **真五行(木火土金水)+ 心**。生克采用经典五行口诀;心系中立,不参与生克(规则详见 [wuxing-reference](wuxing-reference.md) 与第 6 章)。

字怪同样带属性,其弱点即五行相克的部首——这让"凑字"有了明确的针对目标(详见第 8 章)。

## 3.5 一个回合的流程

1. **回合开始**:玩家获得 3 AP;部件池 +2 随机部件(池满则不掉;随机权重为待设计项);回合开始触发的效果结算(如「润泽」回血)。
2. **玩家行动阶段**:玩家自由使用 AP,任意顺序组合拆 / 合 / 出字,直到 AP 耗尽或主动结束回合。
3. **回合结束结算**:玩家这侧的回合末效果结算——灼烧伤害、心魔自伤、裂甲、护盾衰减等,**按固定顺序**结算(见 3.7)。
4. **敌人行动**:每只字怪依次行动(攻击、施加效果、特殊能力)。
5. **敌人回合末结算** → 回到步骤 1。

---

## 3.6 核心张力分析

整个战斗设计的张力,集中在两个决策上:

**张力一:拆 vs 留。**
一个现成的强字是即战力,但把它拆掉能得到灵活的部件,去够到你 build 真正想要的字。散件是"流动性",完整字是"已实现的价值"。玩家必须不断判断:手上这个字,留着打出去价值高,还是拆成零件价值高?

**张力二:攒 vs 出。**
3 点 AP 是稀缺的。这回合全花在拆/合上(攒资源、为下回合的大招铺垫),还是花在出字上(立即输出)?攒得太久会被敌人压死,出得太急则 build 永远成型不了。

这两个张力贯穿每一个回合,是《字·斗》区别于普通卡牌游戏的玩法核心。

---

## 3.7 战斗结算顺序

为保证可预测性,回合末结算严格按固定顺序执行:

1. 灼烧伤害(火)
2. 裂甲(金,护甲衰减)
3. 心魔自伤(心)
4. 草木单位生长(木)
5. 增益的衰减与到期(**护盾例外**:护盾的全清结算点在**敌方行动结束后**,否则护盾永远挡不到攻击,见 10.2)
6. 触发"回合末"的法宝效果

伤害与状态变化在同一步内同时结算,避免顺序歧义。

## 3.8 关键规则细则

这一节集中明确若干高频被问及、但容易模糊的规则。

### 3.8.1 字库容量与字的复用

- **字库基础容量**:6 张(v0.5.1;局内看广告一次性 +2,关卡结束恢复;可被法宝调整,如「作茧自缚」字库上限 −2)。字库达上限后,新合成或获得的字必须替换掉一张已有字,或选择不要。
- **字的复用规则**:出字后,该字进入"已使用"状态,**本场战斗不再可用**,但**战斗结束后回归字库**(下场战斗可重新使用)。这是"一场战斗内的资源管理",不是永久消耗。
- 例外:某些奇遇 / 法宝可以让你"消耗一个字"——这是永久移除,有明确提示。

### 3.8.2 部件池容量

- **部件池容量**:10 个(v0.5.1;局内看广告一次性 +2,关卡结束恢复;可被法宝调整)。达上限后再拾取需先丢弃或合掉。
- 部件池**跨战斗保留**——它是一局 run 内的持久资源,run 结束清空。

### 3.8.3 索敌规则

- **单体伤害字**:默认弹出目标选择,玩家点击敌人确认;长按可锁定默认目标。
- **AOE / 治疗 / 自身增益**:无需选目标。
- **特殊字**(如「怒」需要指定两个敌人):有明确的二次选择提示。

### 3.8.4 失败条件(v0.3)

- 玩家生命降至 0 即败北。
- **死亡即结算 run**,无复活机制。Premium 模式下没有广告复活,死亡是肉鸽的纯粹性,Steam 玩家欣赏这一点。
- 失败也累积字典图鉴 / 字怪图鉴 / 难度阶梯尝试记录("废稿也是稿"),不挫败。详见第 15 章 v0.3。

## 3.9 完整战例:燎原流"焚"连招

> 演示三动词如何串成一套打法。玩家走燎原流(火)。

**初始**:玩家持有「灯」(小灼烧字);场上 2 只木属性杂兵(各 12 血)。

**回合(3 AP)**

1. **拆「灯」**(1 AP)→ 灯 = 火 + 丁,得到部件「火」「丁」入池。
2. 玩家此前已拾得部件「木」「木」。**合「林」**(1 AP)→ 木+木 → 林 入字库。
3. **合「焚」**(1 AP)→ 林+火 → 焚 入字库。

> AP 用尽。本回合未直接造成伤害,但完成了"拆 → 合 → 合"的升阶链。

**下一回合**:出「焚」(2 AP)→ 全体 18 火伤 + 每个被烧到的敌人挂灼烧 → 两只杂兵同时被清。

这一例展示了核心循环:**一个不起眼的「灯」,经拆解、拾取、两次合成,变成了一个清场大招「焚」。** 这就是《字·斗》战斗的爽点所在。
