## 7.1 设计原则

### 原则一:法宝即成语

法宝的本质是**一条被动的规则修正器**。而成语本身大多在描述一条"做事的法则"。两者天然同构,因此法宝**几乎全部用成语命名**——玩家光看名字就能猜到效果,**零学习成本**。这是《字·斗》独有的命名便宜。

### 原则二:变现安全线

法宝可以很强,但 Premium 买断制下(第 14 章 v0.3)**法宝全部靠游玩获取**,不存在任何付费获取途径;DLC 只新增横向内容(新法宝池、新流派),不卖力量。

---

## 7.2 稀有度五档

| 稀有度 | 获取途径 | 强度定位 |
|---|---|---|
| 普通 | 战斗常规掉落、商店 | 稳定的小修正 |
| 稀有 | 战斗、商店 | 中等修正,有 build 倾向 |
| 传说 | 战斗(低概率)、商店 | 强力,常是 build 核心件 |
| 秘宝 | 仅 Boss 掉落 | 强力且独特 |
| 诅咒 | 仅奇遇事件换取 | 强力但带明确代价 |

---

## 7.3 获取方式

- **战斗奖励**:普通战后三选一,可能出现普通/稀有法宝。
- **字摊(商店)**:用金钱购买,价格随稀有度上升。
- **Boss 掉落**:击败 Boss 必得一件秘宝。
- **奇遇事件**:诅咒法宝只能从奇遇事件中"交易"获得(如「讹的低语」)。
- **藏书阁奇遇**:三选一拿一件法宝。

## 7.5 法宝与流派的 synergy

每件法宝都有明确的 synergy 钩子,与某条流派或跨属性组合呼应:

- 属性专精类直接服务对应流派(星火燎原 → 燎原流;锋芒毕露 → 锋锐流)。
- 拆合类服务所有 build,但「点石成金」尤其奖励探索型玩家。
- 诅咒类提供"高风险高回报"的 build-around 节点,服务高难度玩家。
- 玩家在战后三选一时,会基于"我的 build 想要什么"做选择——这正是 build 构筑的乐趣来源。

## 7.6 法宝的运行规则

### 7.6.1 槽位上限

- 一局之内,玩家可同时携带的法宝**无固定上限**(类似遗物收集模式)——只受获取节奏限制。
- 但有**诅咒法宝并存上限**:同一局最多携带 2 件诅咒法宝,避免诅咒堆叠失控。

### 7.6.2 同件叠加

- 默认**同一件法宝在一局内只能持有一件**;再次获取时给予替代奖励(相同稀有度的另一法宝或货币)。
- 少数法宝在描述中明确标注"可叠加",方可重复携带(例如「百步穿杨 · 单体伤害 +X」,叠加则 +2X)。

### 7.6.3 一次性消耗型

少数法宝是**一次性消耗**(如「未雨绸缪 · 进入 Boss 战前免费合一个字」用完即弃),描述会明确"消耗后失效",与持续被动型法宝在 UI 上以不同框区分。

> **v0.4 删除**:原 7.6.4「与 v0.2 法宝升级的接口」随养成系统取消而移除——Premium 模式下法宝无升级,一局之内即完整强度。

---

## 7.7 扩展性

法宝系统的内容扩展几乎无限:沿六大类继续填成语即可。后续 DLC 与免费更新可持续新增法宝(配合 DLC 主题,如讹字篇的诅咒法宝池),作为长线内容。法宝名也可在不同书法皮肤主题下复用,形成收藏维度。
