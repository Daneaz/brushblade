# 《字·斗》章节设计文档 · 第 17 章:MVP 范围与验证计划

> 版本 v0.3 · 设计阶段
> 配套总文档:《字·斗》GDD v0.1
> **v0.3 变更(2026-05-26)**:发行平台改为 Steam + 海外移动端;首发 MVP 改为 Steam 优先 + Steam Playtest 测试机制;变现回退到 Premium 买断后,「养成 / 抽卡 / 月卡」相关 MVP 阶段全部删除;首发内容承诺需要诚实匹配定价。

---

## 17.1 核心原则

整套设计涵盖五流派、35 法宝、字怪体系、成语 Boss、拆合引擎、美术、音效……**一个人绝不能一次全做。** MVP 的纪律,就是防止开发者淹死在自己的设计里。

> MVP 不是"小号的成品",而是**"验证一个核心问题的最小东西"**。

《字·斗》的核心问题只有一个:

> **拆汉字、合汉字来战斗,真的好玩吗?**

五章、五流派、变现、四端——全部是次要的。如果拆/合这个循环不好玩,其余一切都白搭。所以 MVP 必须**冷酷地只服务这一个问题**。

---

## 17.2 分阶段验证

每一阶段都便宜,每一阶段都在为下一阶段排雷。

| 阶段 | 内容 | 验证问题 | 验证渠道 |
|---|---|---|---|
| **0 · 纸面原型** | 几乎零成本,在写任何代码之前。打印剪卡,在桌上用手玩"拆 / 合 / 出字 / 战斗"循环 | 核心决策(拆 vs 合 vs 出字)本身有趣吗? | 自己 + 2~3 个朋友 |
| **1 · 核心战斗原型** | 真正的 MVP 内核。一段可玩战斗(3~5 场连战),单流派(燎原 / 火),拆合引擎 + 提示系统。不上平台,引擎里能跑即可 | 战斗玩 5~10 分钟好玩吗?玩家不解释能看懂拆/合吗? | 5~10 个朋友 / 同好,本地分发 exe |
| **2 · 单局 roguelike 原型** | 加上肉鸽结构:5×5 地图、节点、三选一、3 条流派、一个成语小 Boss | 一局 15~25 分钟体验成立吗?玩家想"再来一局"吗?build 涌现有惊喜感吗? | 20~30 人 itch.io 内测 / 关系群封测 |
| **3 · Steam Playtest 公测** | 内容铺到 17.3 范围,Steam 商店页同步上线开放 Wishlist | 真实玩家手感、留存、退款率信号、Wishlist 增长曲线 | Steam Playtest + TestFlight + Google Play 内部测试,目标 200~1000 测试者 |
| **4 · 正式上架** | 首发上市 | 首月销量、好评率、退款率、口碑传播 | Steam / iOS / Android 公开发售 |

### go / no-go 闸门

每个阶段之间是一道闸门:

- 阶段 1 如果核心战斗不好玩,就停下或转向——**绝不带着没验证的核心往下做**。
- **阶段 3 Steam Playtest 期间如果好评率 < 70%、或退款信号过强,延后正式上架,先迭代修正**。这是 Premium 模式下最重要的一道闸——首发声量是不可逆的。
- 阶段 4 上架后**首周观测**:Mostly Positive 70%+ 是底线,Very Positive 80%+ 才有长尾。

> 个人开发最大的死法,就是把所有东西做完才发现核心不好玩。
> Premium 模式下还有第二种死法:首发那一刻没有 5000+ Wishlist,Steam 算法不给推荐位,埋没。

## 17.3 首发 MVP 范围(v0.3)

Premium 模式下,首发就是「正式发售版本」,玩家付费换"完整体验"。MVP 范围必须**既能验证核心、又能撑起 $12.99 定价的内容承诺**。

### 做(IN)

**玩法核心**

- 拆合引擎 + 合字提示系统。
- **3 条流派**:燎原(火)/ 江海(水)/ 锋锐(金)——爆发、续航、狙杀三种迥异手感。
- 约 200~300 字 + 100~150 部件(比 v0.2 数字稍微提升,因为没有养成"刷字解锁"了,首发要给足初始内容)。
- **3 章**(蒙学 / 字林 / 词渊),各一个成语小 Boss,第三章 Boss 暂作收尾大 Boss。
- 20~25 件法宝(35 件的子集,比 v0.2 略多)。
- roguelike 地图 + 三选一 + 中量级奇遇事件(10~15 个)。
- 新手引导(导演过的首局,详见第 11 章 v0.3)。
- 基础难度 + 难度阶梯 1~5 级。
- **字典图鉴**(已正过的字 + 已收服的字怪 + 已收集的法宝)。
- **Steam 成就 10~15 个**(首次通关、单流派通关、未死亡通关等)。

**平台**

- **Steam(Windows + Mac)主战场**,首发即上;**iOS / Android 同步或滞后 1~3 个月**。

### 不做(OUT,留作 DLC 或后续更新)

- 森罗流、摄心流;10 条跨属性组合(留作 DLC 1「流派扩展包」)。
- 终 Boss「讹」完整体与第 4、5 章(留作 DLC 2「讹字篇」,见第 14 章 v0.3 的 DLC 路线)。
- **难度阶梯高层(6 级以上)**——首发只到 5 级,后续免费更新逐步扩到 20+ 级,保留长尾。
- 每日挑战、排行榜(若做,只用 Steamworks Leaderboards 简易版,不自建后端)。
- UGC、Steam Workshop(留作长尾免费更新)。
- 节日主题字怪皮肤(留作 DLC 3「节日包」)。

> **关键判断**:Premium 首发不能像 v0.2 那样"先开最小变现 + 留存验证后再加养成"——Premium 玩家一次付费就要看到完整体验。**首发体量需要诚实匹配 $12.99 价位**,如果做不到 3 流派 + 3 章 + 5 难度阶梯,**就降价首发到 $7.99(Early Access)而不是涨内容承诺**(见第 14 章 v0.3 的 Early Access 备选方案)。

## 17.4 关于"三端"的取舍(v0.3)

目标三端(Steam PC/Mac、iOS、Android)。引擎一套代码三端导出,"跨端就绪"从第一天成立。

**发布顺序**:**Steam 优先,移动端随后**。

- **Steam 是 indie 卡牌肉鸽的主战场**,Wishlist 机制、Steam Playtest、Steam Cloud、Steamworks 成就/排行榜全套免费,商店页是唯一能持续积累的"未来曝光资产"。
- **移动端是 Steam 的口碑延伸**,Slay the Spire 和 Balatro 都是 PC 验证完再移动端收割长尾。这给了你:(1) 时间补移动端触屏 UI 的特殊问题 (2) PC 评测口碑作为移动端商店页的背书 (3) 错峰宣发,二次声量。

> 这不是放弃平台,只是把风险错开。同步发布并非不可能,但对单人开发是过劳负担。

## 17.5 验证指标(v0.3)

### 阶段 0~2(定性为主)

看真人玩。关键信号:

- 不用解释能否看懂拆/合?
- 会不会笑?会不会主动想继续 / 再来一局?
- 记录所有"卡住"的瞬间(同时为新手引导排雷)。

### 阶段 3 Steam Playtest(定性 + 部分定量)

| 指标维度 | 关键指标 | 健康参考 |
|---|---|---|
| **Wishlist 增长** | Steam 商店页周新增 Wishlist 数 | 上线第一周 200+,稳定期周 50~100;**正式上架前累计 5000+ 为入门门槛** |
| **完成度** | 新手引导完成率、首局通关率、首通章节分布 | 引导完成 80%+,首局通关 30~50% |
| **玩法健康** | 流派分布(3 条是否都有人玩)、流派胜率、平均局时长 | 流派分布偏离均衡 ±15% 内 |
| **试玩反馈** | Steam Playtest 讨论区好评 / 差评分布、bug 报告 | 主动正反馈 / 主动负反馈比 ≥ 3:1 |
| **定性** | 差评关键词、卡点反馈 | 同类差评 ≥ 3 条视为信号,要处理 |

### 阶段 4 正式上架(看数据)

| 指标维度 | 关键指标 | 生死线 |
|---|---|---|
| **首周销量** | Steam 首日 / 首周销量 | 首日 200+,首周 1000+ 是个人 indie 的合理起步 |
| **好评率** | Steam Reviews(Mostly Positive / Very Positive) | **Mostly Positive 70%+ 是底线**,Very Positive 80%+ 才有长尾 |
| **退款率** | Steam 退款率(2h / 14d 窗口) | **< 10% 健康,>15% 危险**——是新手引导 / 内容承诺出问题的强信号 |
| **游玩时长分布** | 平均 / 中位时长,长尾比例 | 中位 5h+ 是 Steam 玩家觉得"值"的下限;长尾 20h+ 玩家比例反映难度阶梯吸引力 |
| **传播** | KOL 覆盖数、Reddit / Twitter 提及度 | 自然分享是 Premium 模式的核心增长引擎 |

> Premium 模式下**没有"留存"指标**。玩家通关后不来 = 正常;通关后还来刷难度阶梯 = 加分。不要用 F2P 的留存框架套 Premium。

## 17.6 每阶段的 Claude Code 分工

- **阶段 0**:纯设计,无代码。
- **阶段 1~2**:Claude Code 加速最猛的部分——拆合引擎、战斗结算、提示算法、地图生成全是纯逻辑。Opus 定数据模型与核心算法,Sonnet 批量实现与填字表。
- **人工**:场景搭建、UI 布局、美术接入仍是编辑器里的人工活;数值平衡反复迭代。

---

## 17.7 一句话总结(v0.3)

> 首发 = 3 流派 / 3 章 / 5 级难度阶梯 / Steam 优先 / Premium $12.99 一次买断,分五阶段验证、阶段间设 go-no-go 闸门;先证明"拆合战斗好玩",再用 Steam Playtest 验证内容承诺与定价匹配,最后正式上架收口碑。

## 17.8 与其他系统的关系

- **拆合引擎(第 4 章)**:阶段 0~1 的核心验证对象。
- **流派 / 数值(第 5、10 章)**:首发取 3 条流派。
- **技术架构(第 16 章 v0.3)**:Steam 优先,引擎选型在阶段 0 收口。
- **变现(第 14 章 v0.3)**:Premium 一次性买断,首发即正式版,不分变现阶段。
- **留存(第 15 章)**:Premium 模式下"留存"的承载从月卡/赛季改为 DLC + 难度阶梯 + 免费更新。
